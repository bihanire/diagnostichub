# Branch Officer UAT Script

## Purpose

Use this script during the pilot to confirm the app is clear, fast, and useful for real branch work.

## Before you begin

- Use a phone or low-end laptop if possible.
- Open the app home page.
- Keep one real or realistic sample issue ready.
- Use normal language, not technical terms, when searching.

## Test 1: Free-text search

Type this:

`phone not turning on but vibrates`

Expected result:

- The app suggests `Phone Not Powering On`
- The next step is easy to understand
- The page is fast and readable

Ask the tester:

- Was the suggested flow sensible?
- Was the wording easy to understand?

## Test 2: Quick-access issue

Tap:

- `Screen Issue`

Expected result:

- The app loads a matching issue flow quickly
- The officer understands what the app is asking next

Ask the tester:

- Did the quick button help?
- Would you know what to do next without help?

## Test 3: Guided triage flow

Start a triage flow and answer the yes/no questions.

Expected result:

- Only one question shows at a time
- Buttons are easy to tap
- Progress is visible
- Questions are short and clear

Ask the tester:

- Did any question feel confusing?
- Did you know why the app was asking that question?

## Test 4: Result screen clarity

Reach a final result.

Expected result:

- Diagnosis is understandable
- Immediate action is clear
- Customer script is easy to read aloud
- Related actions are helpful, not overwhelming

Ask the tester:

- Would you feel confident saying this to a customer?
- Is anything unclear or too technical?

## Test 5: Resume a saved case

Start a flow, leave it midway, then return to the home page.

Expected result:

- The app shows the saved case
- The tester can continue or clear it

Ask the tester:

- Is the saved-case message clear?
- Would this help in a busy branch situation?

## Test 6: Feedback submission

On the result page, submit feedback.

Expected result:

- The feedback action is easy to find
- The branch label is optional
- The success message is clear

Ask the tester:

- Would you actually use this feedback option after a difficult case?

## Test 7: Slow-network feel

Refresh the page or repeat a search under typical branch connectivity.

Expected result:

- The app still feels responsive
- Loading states are understandable
- The user never feels lost

Ask the tester:

- Did the app feel slow?
- If it paused, did you still understand what was happening?

## Test 8: Wrong or unclear match

Type a messy issue that may not match well, for example:

`customer says phone has strange problem after water and screen dark`

Expected result:

- The app gives a best match or fallback guidance
- The result does not feel broken
- The user still knows what to do next

Ask the tester:

- If the app is unsure, does it still help?

## What the facilitator should record

- Search text used
- Procedure suggested
- Where the tester hesitated
- Questions that were misunderstood
- Whether the final action felt correct
- Whether the customer script felt natural
- Any comment the tester would want changed before rollout

## UAT sign-off questions

At the end, ask the tester:

1. Would you use this during a real customer interaction?
2. Was anything too technical or too long?
3. Did the recommendation feel trustworthy?
4. What is the first thing you would change?

## Pass criteria

The pilot case passes if:

- the tester can complete the flow without coaching
- the recommendation is understandable
- the customer script is usable
- the tester says the tool would help in real branch work
